# Guía para agentes

## Objetivo del repositorio

Construir la landing pública de Entrenemos. No duplicar aquí la aplicación mobile ni el panel web autenticado.

## Antes de implementar

1. Leer `README.md` y toda la carpeta `docs/`.
2. Revisar visualmente `logo.png` y `logo2.png`.
3. Consultar `../EntrenemosWeb/src/app/globals.css` para los tokens vigentes y `../GymApp/gym_app/lib/theme/` para validar coherencia con mobile.
4. No modificar los repositorios vecinos desde este proyecto.
5. Confirmar decisiones todavía abiertas del brief; no inventar precios, testimonios, métricas, enlaces de descarga ni claims médicos.

## Criterios base

- Español rioplatense claro, directo y cercano; evitar lenguaje agresivo de fitness.
- Diseño dark, premium, humano y enérgico.
- Mobile-first, accesible, rápido y responsive.
- Usar `#0D93F2` como acento principal y respetar contraste AA.
- El mensaje debe incluir tanto a atletas como a entrenadores.
- Tratar “Entrenemos” como una invitación compartida: progreso con acompañamiento, constancia y datos útiles.
- Mantener SEO, metadata social, favicon, estados de foco, reducción de movimiento y optimización de imágenes como requisitos, no extras.

## Fuentes de verdad

- Marca y lineamientos de landing: documentación de este repositorio.
- Producto mobile: `../GymApp/gym_app`.
- Producto web y tokens CSS: `../EntrenemosWeb`.
- Descripción pública y URL prevista: `../EntrenemosWeb/src/lib/config/seo.ts`.

Si los productos vecinos cambian, actualizar primero esta documentación y dejar registrada la diferencia.
